# Как использовать документацию

## Для AI ассистента

### Перед началом работы

**Всегда начинай с чтения этих файлов по порядку:**

1. **CURRENT_STAGE.md** (в корне проекта)
   - Текущее состояние проекта
   - Последние изменения
   - Текущие задачи
   - Референсы на важные файлы

2. **PHILOSOPHY.md** (в корне проекта)
   - Философия работы над проектом
   - Процесс работы
   - Принципы разработки
   - Что НЕ делать

3. **IMPORTANT_NOTES.md** (в корне проекта)
   - Критически важные правила
   - Информация о сервере
   - Обработка ошибок

### При работе с конкретной задачей

Используй модульную документацию в `.cursor/docs/`:

- **error-handling.md** - при работе с обработкой ошибок
- **user-ids.md** - при работе с ID пользователей
- **dates-time.md** - при работе с датами и временем
- **cron-jobs.md** - при работе с cron задачами

### Команда для начала работы

Когда пользователь дает задачу, используй эту команду:

```
Перед тем, как начать работу, прочитай CURRENT_STAGE.md, PHILOSOPHY.md и IMPORTANT_NOTES.md по порядку.

Затем выпиши кратко наши задачи и я дам команду для следующего шага.
```

## Для разработчика

### Обновление CURRENT_STAGE.md

После значительных изменений обновляй `CURRENT_STAGE.md`:
- Последние изменения
- Текущие задачи
- Известные проблемы

### Добавление новой документации

При добавлении нового аспекта проекта:
1. Создай файл в `.cursor/docs/`
2. Название: `kebab-case.md`
3. Добавь ссылку в `.cursor/docs/README.md`
4. При необходимости добавь референс в `CURRENT_STAGE.md`

## Структура документации

```
.
├── CURRENT_STAGE.md          # Текущее состояние (читать первым)
├── PHILOSOPHY.md             # Философия работы
├── IMPORTANT_NOTES.md        # Критически важные правила
├── .cursorrules              # Правила для AI
└── .cursor/
    └── docs/
        ├── README.md         # Обзор модульной документации
        ├── HOW_TO_USE.md     # Этот файл
        ├── error-handling.md # Обработка ошибок
        ├── user-ids.md       # ID пользователей
        ├── dates-time.md     # Даты и время
        └── cron-jobs.md      # Cron задачи
```
