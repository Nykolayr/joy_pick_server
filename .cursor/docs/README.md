# Документация проекта Joy Pick Server

Это модульная документация для AI ассистента. Каждый файл описывает отдельный аспект проекта.

## Структура документации

### Основные документы
- **[CURRENT_STAGE.md](../../CURRENT_STAGE.md)** - текущее состояние проекта (читать в первую очередь)
- **[PHILOSOPHY.md](../../PHILOSOPHY.md)** - философия работы над проектом
- **[IMPORTANT_NOTES.md](../../IMPORTANT_NOTES.md)** - критически важные правила
- **[.cursorrules](../../.cursorrules)** - правила для AI ассистента

### Модульная документация (в этой папке)

#### [error-handling.md](./error-handling.md)
Обработка ошибок в API:
- Использование функции `error()`
- Структура ответа об ошибке
- Примеры обработки различных типов ошибок

#### [user-ids.md](./user-ids.md)
Работа с ID пользователей:
- Использование UUID из БД (не Firebase UID)
- Валидация UUID
- Проверка существования пользователей

#### [dates-time.md](./dates-time.md)
Работа с датами и временем:
- Формат дат (UTC ISO 8601)
- Нормализация дат
- Поля с датами в моделях

#### [cron-jobs.md](./cron-jobs.md)
Cron задачи и автоматические действия:
- Список всех cron задач
- Логика каждой задачи
- Логирование и просмотр истории

#### [stripe-admin-api.md](./stripe-admin-api.md)
Stripe Admin API для администрирования платежей:
- Получение активных заявок с платежами
- Получение закрытых заявок с финансовой информацией
- Структура ответов и статусы

## Как использовать

1. **Перед началом работы:** Прочитать [CURRENT_STAGE.md](../../CURRENT_STAGE.md)
2. **При работе с конкретной задачей:** Прочитать соответствующий файл из этой папки
3. **При сомнениях:** Обратиться к [PHILOSOPHY.md](../../PHILOSOPHY.md)
4. **Подробная инструкция:** См. [HOW_TO_USE.md](./HOW_TO_USE.md)

## Добавление новой документации

При добавлении нового аспекта проекта создайте новый файл в этой папке:
- Название файла: `kebab-case.md`
- Структура: заголовки, примеры кода, ссылки на связанные документы
- Референсы: ссылки на другие документы, где это уместно
